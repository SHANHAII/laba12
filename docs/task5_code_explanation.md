## Объяснение функции `calculate_transfer_fee`

### Что делает функция целиком

Функция считает, сколько денег банк возьмёт в качестве комиссии за перевод.
На входе — сумма, тип счёта, валюта и флаг премиум-клиента.
На выходе — размер комиссии, округлённый до копеек.

### Разбор по блокам

```python
if amount <= 0:
    raise ValueError(...)
```
**Что делает:** Проверяет, что сумма перевода положительная.
**Зачем:** Нельзя взять комиссию с нулевого или отрицательного перевода — это ошибка входных данных, поэтому выбрасывается исключение, а не возвращается -1 (как в плохом коде).

```python
if currency == "RUB":
    fee = _get_rub_fee(amount, account_type)
else:
    fee = _get_foreign_fee(amount, account_type)
```
**Что делает:** Выбирает алгоритм расчёта в зависимости от валюты.
**Зачем:** Рублёвые переводы имеют прогрессивную шкалу ставок (чем больше сумма — тем выше %), а для иностранных валют фиксированный процент. Разделение по функциям позволяет легко добавить новые валюты без изменения основной логики.

```python
RATES_RUB = {
    "checking": [(1_000, 0.010), (50_000, 0.015), ...]
}
```
**Что делает:** Таблица ставок — список пар (порог суммы, процент).
**Зачем:** Вместо цепочки `if/elif` используется цикл: перебираем пороги по порядку и берём ставку первого подходящего. Это устраняет дублирование кода и упрощает добавление новых диапазонов.

```python
return max(amount * rate, MIN_FEE_RUB[account_type])
```
**Что делает:** Берёт максимум из расчётной комиссии и минимальной комиссии.
**Зачем:** При маленьких суммах вычисленный процент может быть меньше минимума (например, 1% от 500₽ = 5₽, но минимум — 10₽). `max()` решает это в одну строку вместо `if x < min: x = min`.

```python
if is_premium:
    fee *= PREMIUM_DISCOUNT
```
**Что делает:** Применяет скидку 10% для премиум-клиентов.
**Зачем:** Простое умножение после расчёта базовой комиссии. Константа `PREMIUM_DISCOUNT = 0.9` документирует смысл числа.

```python
return round(fee, 2)
```
**Что делает:** Округляет до 2 знаков после запятой.
**Зачем:** Финансовые операции должны работать с точными суммами — дробные копейки недопустимы.
